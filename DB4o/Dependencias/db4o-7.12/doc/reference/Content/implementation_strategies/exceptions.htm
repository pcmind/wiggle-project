Exceptions

There are many situations when it is vitally important for the user to know what went wrong and to be able to react accordingly. Some of the examples:

  • callbacks;
  • opening database file or client connection;
  • unsupported schema changes;
  • invalid ID format;
  • corrupted database file, etc

The db4o allows exceptions to "bubble up" to the user level.
